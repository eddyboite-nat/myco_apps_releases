\begin{abstract}
Ce document présente, selon une structure IMRaD stricte, la conception fonctionnelle d'un
pipeline R complet pour évaluer automatiquement trois indicateurs écologiques par pelouse :
le potentiel fongique, l'intérêt patrimonial et le gradient CHEGD avec indice de
représentativité IR. Les méthodes décrivent l'ingestion, la normalisation, les calculs métier
et le module statistique de fiabilité. Les résultats formalisent les artefacts produits
(CSV, figures, synthèses et contrôles QA). La discussion précise l'interprétation, les limites
opérationnelles et les conditions de réutilisation du pipeline.
\end{abstract}

\subsection{À propos}

Ce document décrit la spécification fonctionnelle complète du script :

\begin{center}
  \texttt{scripts/Evaluation\_Potentiel\_Fongique\_Interets\_Patrimoniaux\_CHEGD.R}
\end{center}

Le script a été conçu pour automatiser l'évaluation mycologique de pelouses en s'appuyant sur
la méthode CHEGD (\textbf{C}lavaria, \textbf{H}ygrocybe/Cuphophyllus, \textbf{E}ntoloma,
\textbf{G}eoglossaceae, \textbf{D}ermoloma), qui constitue un indicateur de référence pour
l'évaluation de la valeur patrimoniale des prairies en Europe occidentale.

\subsection{Objectifs du pipeline}

Le pipeline réalise automatiquement, pour chaque site (pelouse) :

\begin{enumerate}[label=\arabic*.]
  \item Lecture et validation d'un fichier de données brutes CSV.
  \item Production de résumés descriptifs globaux (par site, famille, espèce, date, fiabilité).
  \item Calcul du \textbf{potentiel fongique} : score pondéré par groupe fonctionnel.
  \item Calcul de l'\textbf{intérêt patrimonial} : indice CHEGD et classification en 5 niveaux.
  \item Calcul du \textbf{gradient CHEGD} : nombre d'espèces CHEGD par visite.
  \item Calcul de l'\textbf{Indice de Représentativité (IR)} par visite et par site.
  \item Génération de 7 figures de synthèse (PNG 300 dpi + PDF vectoriel).
  \item Module statistique : analyse de la fiabilité des déterminations (4 objectifs).
  \item Export de tous les résultats en fichiers CSV encodés UTF-8.
\end{enumerate}

\subsubsection{Résolution des chemins}

Le script résout automatiquement les chemins relatifs selon l'environnement d'exécution,
par ordre de priorité :

\begin{enumerate}[label=\arabic*.]
  \item Argument \texttt{--file=<chemin>} injecté par \texttt{Rscript}.
  \item \texttt{sys.frames()[[1]\$ofile]} : injecté par \texttt{source()} ou RStudio.
  \item \texttt{getwd()} : répertoire de travail courant (fallback VSCode / console interactive).
\end{enumerate}

Le fichier d'entrée est résolu via une stratégie à \textbf{3 niveaux} :

\begin{enumerate}[label=\arabic*.]
  \item \textbf{Chemin exact} : \texttt{file.exists()} sur le chemin résolu.
  \item \textbf{Correspondance normalisée} : comparaison après suppression des accents, mise en minuscules et retrait des caractères non alphanumériques.
  \item \textbf{Distance de Levenshtein} (\texttt{adist}) : candidat le plus proche si la distance est inférieure au seuil $\max(3,\, \lfloor 0{,}25 \times \text{long}(\text{nom})\rfloor)$.
\end{enumerate}

\subsubsection{Nettoyage et enrichissement}

Après lecture, les colonnes suivantes sont ajoutées au tableau brut :

\begin{itemize}
  \item \texttt{date\_obs} : conversion robuste de la colonne date vers le type \texttt{Date}.
  \item \texttt{nombre\_espece\_num} : conversion de l'effectif en numérique, avec tolérance de la virgule décimale.
\end{itemize}

\subsubsection{Normalisation de texte}

La fonction \texttt{normalize\_text()} est utilisée systématiquement avant toute comparaison de noms d'espèces ou de sites. Elle applique, dans l'ordre :

\begin{enumerate}[label=\arabic*.]
  \item Conversion en chaîne de caractères.
  \item Translittération ASCII (\texttt{iconv} avec \texttt{ASCII//TRANSLIT}).
  \item Conversion en minuscules.
  \item Remplacement des caractères non alphanumériques par un espace, puis contraction des espaces multiples.
\end{enumerate}

\subsubsection{Référentiel des sites}

La fonction \texttt{prepare\_site\_reference()} construit un référentiel continu de sites de 1 jusqu'au plus grand identifiant détecté :

\begin{itemize}
  \item Les identifiants sont extraits avec la regex \texttt{\\d+} depuis les valeurs textuelles (ex : «~Pelouse 16~» $\to$ 16).
  \item Les sites non observés dans les données reçoivent un nom synthétique «~Pelouse N~».
  \item Le référentiel garantit que les sites intermédiaires manquants sont inclus dans toutes les métriques.
\end{itemize}

% ============================================================================================================================
\subsection{Calcul des métriques par site}
% ============================================================================================================================

\subsubsection{Principe général}

Les trois indicateurs sont calculés dans \texttt{build\_site\_level\_metrics()} à partir du
tableau dédupliqué des espèces par site (\texttt{distinct(site\_id, species\_raw, species\_norm,
genus, family\_norm)}). Seules les \textbf{espèces uniques par site} contribuent aux scores
(pas de double comptage par visite).

\subsubsection{Potentiel fongique}

\medskip
\noindent\textbf{Définition.}\par

Le potentiel fongique est un \textbf{score de richesse pondérée} par groupe fonctionnel CHEGD.
Il reflète la valeur écologique globale du site en additionnant les contributions de chaque
groupe d'espèces indicatrices, pondérées selon leur rarité et leur valeur diagnostique.

\medskip
\noindent\textbf{Grille de pondération.}\par

\begin{table}[htbp]
  \centering
  \caption{Pondération des groupes fonctionnels pour le score de potentiel fongique}
  \footnotesize
  \begin{tabularx}{\textwidth}{L{2.6cm}YcL{3.2cm}}
    \toprule
    \textbf{Groupe} & \textbf{Genres / espèces} & \textbf{Poids} & \textbf{Variable} \\
    \midrule
    Mycènes et litière & \textit{Mycena, Galerina, Crinipellis} & $\times$1 & \path{count_litter} \\
    Coprophiles & \textit{Coprinus, Coprinellus, Conocybe, Panaeolus, Psathyrella, Psilocybe, Stropharia, Coprinopsis, Panaeolina} & $\times$1 & \path{count_dung} \\
    Agarics & \textit{Agaricus} & $\times$1 & \path{count_agaricus} \\
    Entolomes & \textit{Entoloma} & $\times$2 & \path{count_entoloma} \\
    Clavarioïdes & \textit{Clavaria, Clavulinopsis, Ramariopsis} + Géoglossacées & $\times$4 & \path{count_clavaria} \\

    \bottomrule
  \end{tabularx}
\end{table}

\medskip
\noindent\textbf{Formule.}\par

\begin{formulebox}[Score de potentiel fongique]
  \[
  \begin{aligned}
    \text{Score}_{\text{potentiel}} ={}&
    n_{\text{litière}} + n_{\text{copro}} + n_{\text{agaricus}}
    + 2n_{\text{entoloma}} + 4n_{\text{clavaria}} + 6n_{\text{C.zollingeri}} \\
    &+ 2n_{\text{Cuphophyllus}} + 2n_{\text{H.conica}}
    + 2n_{\text{hygro.jaunes}} + 2n_{\text{psittacina}} \\
    &+ 3n_{\text{C.pratensis}} + 3n_{\text{H.reidii}}
    + 7n_{\text{hygro.rouges}} + 10n_{\text{H.calyptriformis}} \\
    &+ 3n_{\text{dermoloma}} + 3n_{\text{camarophyllopsis}}
    + 6n_{\text{porpoloma}} + n_{\text{vesses}}.
  \end{aligned}
  \]
  où $n_X$ est le \textbf{nombre d'espèces uniques} du groupe $X$ observées sur le site.
\end{formulebox}

\subsubsection{Intérêt patrimonial}

\medskip
\noindent\textbf{Définition.}\par

L'intérêt patrimonial est basé sur la \textbf{richesse spécifique maximale parmi les 5 groupes
CHEGD}. Il reflète le degré de développement du cortège mycologique prairial indicateur.

\medskip
\noindent\textbf{Groupes CHEGD.}\par

\begin{table}[htbp]
  \centering
  \caption{Groupes fonctionnels pour l'indice patrimonial (méthode CHEGD)}
  \begin{tabularx}{\textwidth}{clX}
    \toprule
    \textbf{Lettre} & \textbf{Groupe} & \textbf{Genres inclus} \\
    \midrule
    C & Clavarioïdes & \textit{Clavaria, Clavulinopsis, Ramariopsis} \\
    H & Hygrocybes   & \textit{Hygrocybe, Cuphophyllus, Gliophorus, Porpolomopsis} \\
    E & Entolomes    & \textit{Entoloma} \\
    G & Géoglossacées & Famille \textit{Geoglossaceae} ; genres \textit{Geoglossum, Microglossum, Trichoglossum} \\
    D & Dermolomes   & \textit{Dermoloma, Porpoloma, Camarophyllopsis} \\
    \bottomrule
  \end{tabularx}
\end{table}

\medskip
\noindent\textbf{Formule.}\par

\begin{formulebox}[Indice patrimonial]
  \[
    I_{\text{patrimonial}} = \max\!\bigl(n_C,\, n_H,\, n_E,\, n_G,\, n_D\bigr)
  \]
  où $n_X$ est le nombre d'espèces uniques du groupe $X$ sur le site.
\end{formulebox}

\medskip
\noindent\textbf{Classification en 5 niveaux.}\par

\begin{table}[htbp]
  \centering
  \caption{Classes d'intérêt patrimonial (inspirées de la méthode CHEGD)}
  \begin{tabular}{clc}
    \toprule
    \textbf{Seuil} & \textbf{Classe} & \textbf{Couleur recommandée} \\
    \midrule
    $I \leq 2$         & Intérêt faible         & vert très clair \\
    $2 < I \leq 5$     & Intérêt local          & vert clair \\
    $5 < I \leq 10$    & Intérêt régional       & vert moyen \\
    $10 < I \leq 14$   & Intérêt national       & vert foncé \\
    $I > 14$           & Intérêt international  & vert très foncé (\texttt{\#006D2C}) \\
    \bottomrule
  \end{tabular}
\end{table}

\subsubsection{Gradient CHEGD et Indice de Représentativité (IR)}

\medskip
\noindent\textbf{Définition du gradient CHEGD.}\par

Le gradient CHEGD est le \textbf{nombre d'espèces CHEGD uniques observées lors d'une visite}.
Il est calculé par site et par visite (identifiée par la date de sortie, en l'absence de
classeur de référence).

Les espèces CHEGD retenues appartiennent aux genres :
\textit{Clavaria, Clavulinopsis, Ramariopsis, Hygrocybe, Cuphophyllus, Gliophorus,
Porpolomopsis, Entoloma, Geoglossum, Microglossum, Trichoglossum, Dermoloma, Porpoloma,
Camarophyllopsis}, ou à la famille \textit{Geoglossaceae}.

\medskip
\noindent\textbf{Métriques dérivées par site.}\par

\begin{table}[htbp]
  \centering
  \caption{Métriques CHEGD exportées par site}
  \begin{tabularx}{\textwidth}{lX}
    \toprule
    \textbf{Colonne} & \textbf{Définition} \\
    \midrule
    \texttt{nb\_visites\_planifiees} & Nombre de visites distinctes (dates) \\
    \texttt{gradient\_visite\_N}     & Nombre d'espèces CHEGD uniques à la visite N \\
    \texttt{chegd\_total}            & Somme des gradients sur toutes les visites \\
    \texttt{chegd\_moyen}            & Moyenne des gradients par visite \\
    \texttt{ir\_visite\_N}           & Indice de représentativité à la visite N \\
    \texttt{ir\_moyen}               & Moyenne des IR par visite \\
    \bottomrule
  \end{tabularx}
\end{table}

\medskip
\noindent\textbf{Formule de l'Indice de Représentativité.}\par

\begin{formulebox}[Indice de Représentativité (IR) par visite]
  \[
    IR_{\text{visite}} = \max\!\left(0,\; 1 - \frac{G_{\text{visite}}}{G_{\text{total}}}\right)
  \]
  où :
  \begin{itemize}
    \item $G_{\text{visite}}$ : gradient CHEGD de la visite considérée,
    \item $G_{\text{total}}$ : nombre total d'espèces CHEGD distinctes sur l'ensemble des visites du site.
  \end{itemize}
  Si $G_{\text{total}} \leq 0$, on pose $IR = 0$ (pas de division par zéro).
\end{formulebox}

L'IR moyen par site est la moyenne arithmétique des IR individuels :
\[
  IR_{\text{moyen}} = \frac{1}{V}\sum_{v=1}^{V} IR_{\text{visite}_v}
\]

\begin{infobox}[Interprétation de l'IR]
  \begin{itemize}
    \item Un IR proche de \textbf{1} indique que le gradient d'une visite est faible par rapport au total du site : la visite apporte peu de nouvelles espèces CHEGD (bon signe de suivi avancé).
    \item Un IR proche de \textbf{0} indique que la visite concentre l'essentiel des espèces CHEGD observées.
    \item L'IR moyen mesure la \textbf{régularité de répartition des espèces CHEGD entre les visites}.
  \end{itemize}
\end{infobox}

\subsubsection{Objectif 3 — Sélection du meilleur modèle prédictif de la fiabilité}

\textbf{But :} Identifier le modèle statistique prédisant le mieux le niveau de fiabilité
d'une observation à partir de ses caractéristiques (site, famille, saison, abondance).

\medskip
\noindent\textbf{Modèles candidats.}\par

\begin{table}[htbp]
  \centering
  \caption{Modèles de classification candidats}
  \begin{tabularx}{\textwidth}{llX}
    \toprule
    \textbf{Identifiant} & \textbf{Modèle} & \textbf{Description} \\
    \midrule
    \texttt{polr} & Régression ordinale (\texttt{MASS::polr}) & Tient compte de l'ordre des classes (Non renseignée < Probable < Certaine). Lien logistique (\texttt{logistic}). \\
    \texttt{multinom} & Régression multinomiale (\texttt{nnet::multinom}) & Classes traitées de façon nominale, sans contrainte d'ordre. \\
    \texttt{baseline\_majority} & Classifieur majoritaire & Prédit systématiquement la classe la plus fréquente. Sert de référence basse. \\
    \bottomrule
  \end{tabularx}
\end{table}

\medskip
\noindent\textbf{Formule des modèles.}\par

\[
  \text{fiabilite} \sim \text{abundance} + \text{season} + \text{site} + \text{family}
\]

Les modalités de site et de famille les moins fréquentes sont regroupées dans une catégorie
\texttt{Autre} (les 12 modalités les plus fréquentes sont conservées) pour éviter la
sur-paramétrisation et les nouvelles modalités sur la partition de test.

\medskip
\noindent\textbf{Protocole d'évaluation.}\par

Validation croisée $k = 5$ folds, avec stratification aléatoire (\texttt{set.seed(42)}).
Conditions minimales : au moins 20 observations et au moins 2 classes de fiabilité distinctes.

\medskip
\noindent\textbf{Métriques d'évaluation.}\par

\begin{table}[htbp]
  \centering
  \caption{Métriques calculées par modèle et par fold}
  \begin{tabularx}{\textwidth}{lX}
    \toprule
    \textbf{Métrique} & \textbf{Définition} \\
    \midrule
    Accuracy & Proportion de prédictions correctes. \\
    Macro-F1 & Moyenne du F1 par classe (précision × rappel harmonique), pondération uniforme. \\
    Log-loss & $-\frac{1}{n}\sum_{i}\log p_i^*$ où $p_i^*$ est la probabilité prédite pour la vraie classe. Borné dans $[10^{-15}, 1-10^{-15}]$. \\
    \bottomrule
  \end{tabularx}
\end{table}

\begin{formulebox}[Macro-F1]
  \[
    F1_{\text{macro}} = \frac{1}{|C|}\sum_{c \in C} F1_c
    \qquad \text{avec} \quad
    F1_c = \frac{2 \times \text{Précision}_c \times \text{Rappel}_c}{\text{Précision}_c + \text{Rappel}_c}
  \]
\end{formulebox}

\textbf{Sélection :} Meilleur macro-F1 moyen sur les 5 folds ; en cas d'égalité, meilleure accuracy.

\textbf{Sorties :} matrices de confusion, probabilités prédites, coefficients du modèle final
ajusté sur l'ensemble des données, 3 figures.

\subsubsection{Objectif 4 — Test d'inférence : dépendance fiabilité × site}

\textbf{But :} Tester statistiquement si la répartition des niveaux de fiabilité est
indépendante du site d'observation.

\medskip
\noindent\textbf{Sélection automatique du test.}\par

\begin{table}[htbp]
  \centering
  \caption{Logique de sélection du test d'inférence}
  \small
  \begin{tabularx}{\textwidth}{Y Y l}
    \toprule
    \textbf{Condition} & \textbf{Test retenu} & \textbf{Identifiant} \\
    \midrule
    Toutes les cellules attendues $\geq 5$ & Chi-2 (\texttt{chisq.test}) & \texttt{Chi2} \\
    Au moins une cellule attendue $< 5$    & Test exact de Fisher      & \texttt{Fisher} \\
    Fisher exact échoue (tableau trop grand) & Fisher avec Monte-Carlo ($B=10000$) & \texttt{Fisher\_simule} \\
    Tous les tests échouent                & Chi-2 de secours          & \texttt{Chi2\_fallback} \\
    \bottomrule
  \end{tabularx}
\end{table}

\medskip
\noindent\textbf{Hypothèse nulle.}\par

\[
  H_0 : \text{La répartition des niveaux de fiabilité est indépendante du site.}
\]

\textbf{Métrique rapportée :} $-\log_{10}(p\text{-value})$ pour faciliter les comparaisons
(valeur élevée = rejet de l'indépendance).

\textbf{Sorties :} \path{stat_obj4_inference_tests.csv} + heatmap site $\times$ fiabilité.

\section{Discussion}
% ============================================================================================================================

La discussion explicite la portée décisionnelle des indicateurs, les limites méthodologiques
et les situations opérationnelles qui nécessitent un contrôle expert.

\subsection{Interprétation des indicateurs}

\subsubsection{Score de potentiel fongique}

\begin{itemize}
  \item \textbf{Score $\leq 10$ (faible)} : cortège CHEGD peu développé ou mal documenté. Effort d'inventaire ou contexte écologique à vérifier.
  \item \textbf{$10 <$ Score $< 30$ (intéressant)} : présence d'espèces indicatrices notables ; site à suivre et à protéger.
  \item \textbf{Score $\geq 30$ (élevé)} : richesse indicatrice remarquable, intérêt écologique fort ; site prioritaire pour la conservation.
\end{itemize}

\begin{warnbox}[Important]
  Le score de potentiel est sensible à l'effort d'échantillonnage et à la saison.
  Un score faible ne signifie pas l'absence d'intérêt, mais peut refléter un inventaire incomplet.
  Toujours croiser avec l'indice patrimonial et le gradient CHEGD.
\end{warnbox}

\subsubsection{Indice patrimonial}

\begin{itemize}
  \item $I \leq 2$ : site peu ou pas développé pour les CHEGD.
  \item $2 < I \leq 5$ : intérêt local, richesse CHEGD modérée.
  \item $5 < I \leq 10$ : intérêt régional, pelouse à forte valeur indicatrice.
  \item $10 < I \leq 14$ : intérêt national, site exceptionnel à conserver en priorité.
  \item $I > 14$ : intérêt international, richesse CHEGD exceptionnelle.
\end{itemize}

\subsubsection{Gradient CHEGD moyen}

Un gradient CHEGD élevé par visite signale une pression de CHEGD constante sur les visites :
la pelouse héberge un cortège CHEGD abondant et régulièrement exprimé. Un gradient faible
peut indiquer soit une faible richesse CHEGD, soit une forte saisonnalité.

\subsubsection{IR moyen}

\begin{itemize}
  \item IR proche de 1 sur toutes les visites : les espèces CHEGD sont bien réparties entre les visites — signe d'un suivi équilibré ou d'une forte richesse totale.
  \item IR faible sur une visite : cette visite concentre une grande partie du pool CHEGD du site.
  \item IR moyen global élevé : bonne représentativité globale du suivi.
\end{itemize}

\subsubsection{Lecture conjointe des figures 1 et 2}

La Figure 1 permet une lecture rapide et comparative par site sur les 3 dimensions.
La Figure 2 révèle les \textbf{profils écologiques} :
\begin{itemize}
  \item Sites en haut à droite : forte valeur patrimoniale et potentiel élevé (sites prioritaires).
  \item Sites en bas à gauche : faible valeur sur les deux axes (à renforcer en effort ou à contextualiser).
  \item Points gros et rouges : gradient CHEGD élevé — sites à enjeu fort de suivi.
\end{itemize}

\subsection{Limites et perspectives}
La portée des indicateurs dépend directement de la qualité taxonomique, de l'effort
d'échantillonnage, de la saisonnalité et de la complétude des visites. Un score faible ne doit
pas être interprété comme une absence d'intérêt écologique sans vérification du contexte de
prospection. Les évolutions prioritaires concernent la validation sur plusieurs campagnes,
l'ajout de diagnostics d'effort d'inventaire et la documentation systématique des changements
méthodologiques dans le journal de versionnement.

\subsection{Dépannage (FAQ)}
% ============================================================================================================================

\begin{description}
  \item[Le script ne trouve pas le fichier d'entrée.]
    Vérifier qu'un fichier \texttt{.csv} est présent dans \texttt{data/}.
    Le nom est résolu par correspondance normalisée : les accents et la casse ne sont pas bloquants.
    Consulter les messages \texttt{info} dans la console pour identifier le fichier retenu.

  \item[Colonnes manquantes à l'exécution.]
    Le script lève une erreur explicite listant les colonnes introuvables.
    Vérifier que les noms de colonnes dans \texttt{get\_embedded\_config()\$columns} correspondent
    exactement aux en-têtes du fichier d'entrée (y compris accents et espaces).

  \item[Aucune espèce CHEGD détectée / gradients tous à zéro.]
    Vérifier que les noms d'espèces sont bien en format binomial latin (\textit{Genre espèce}).
    La normalisation est insensible aux accents mais requiert un espace entre le genre et l'épithète.
    Contrôler dans \path{donnees_brutes_nettoyees.csv} les colonnes \texttt{species\_norm} et \texttt{genus}.

  \item[Le module fiabilité affiche «~insuffisant~».]
    Le module requiert $\geq 20$ observations avec au moins 2 niveaux de fiabilité distincts.
    Si non rempli, les métriques statistiques sont exportées comme \texttt{NA}.

  \item[Les figures sont vides ou absentes.]
    Vérifier que \texttt{site\_metrics\$combined} n'est pas vide.
    Un tableau d'entrée sans identifiant de site numérique (regex \texttt{\\d+})
    produira un référentiel de sites vide : toutes les métriques seront nulles.

  \item[Incohérence CHEGD signalée dans \path{resume_global.csv}.]
    L'indicateur \texttt{coherence\_chegd\_gradients\_ok} vaut 0 si la somme des gradients par
    visite diffère du total CHEGD d'au moins $10^{-9}$. Cela peut survenir si les dates
    d'observation comportent des doublons ou des valeurs manquantes.
\end{description}

\subsection*{Étape 1 — Préparer les données}

Placer le fichier d'observations dans \texttt{data/}, par exemple :

\begin{verbatim}
data/donnees_recoltes_chegd_pelouses.csv
\end{verbatim}

Le script tolère les variantes orthographiques et les accents manquants grâce à la résolution
normalisée (\S\,\ref{sec:config}).

\subsection*{Étape 2 — Configurer (optionnel)}

Modifier la fonction \texttt{get\_embedded\_config()} dans le script pour adapter :
\begin{itemize}
  \item \texttt{input\_file} : chemin vers le fichier d'entrée.
  \item \texttt{columns} : noms de colonnes si le fichier utilise un autre vocabulaire.
\end{itemize}

\subsection*{Étape 3 — Lancer l'analyse}

Depuis la racine du projet :

\begin{verbatim}
Rscript scripts/Evaluation_Potentiel_Fongique_Interets_Patrimoniaux_CHEGD.R
\end{verbatim}

Ou interactivement sous RStudio / VSCode en ouvrant le script et en l'exécutant.

\subsection*{Étape 4 — Consulter les résultats}

Toutes les sorties sont dans :

\begin{verbatim}
results/EPFIP_CHEGD/
\end{verbatim}
