\section*{Vue d'ensemble}
Ce document d'ecrit l'architecture technique du script
\texttt{scripts/Inventaires\_completude\_representativite.R} (v1.4).
Il pr'ecise les contrats d'entr'ee/sortie, la configuration runtime
et les limites op'erationnelles.

\section*{Architecture logique}
\subsection*{Blocs principaux}
\begin{enumerate}
  \item Initialisation et validation de configuration
  \item R\'esolution du fichier d'entr\'ee
  \item Lecture auto-d\'elimiteur
  \item Audit CSV et export de conformit\'e
  \item Pr\'eparation/nettoyage des donn\'ees
  \item Analyse par site
  \item Consolidation multi-sites
  \item Export des livrables CSV/PNG
  \item Manifeste de sortie
\end{enumerate}

\subsection*{Fonctions cl\'es}
\begin{itemize}
  \item \textbf{Configuration et logging} : \texttt{get\_embedded\_config()}, \texttt{setup\_logging()}, \texttt{log\_message()}, \texttt{log\_info()}, \texttt{log\_debug()}, \texttt{log\_warning()}, \texttt{log\_header()}, \texttt{log\_section()}
  \item \textbf{R\'esolution et validation} : \texttt{resolve\_path()}, \texttt{resolve\_input\_file()}, \texttt{validate\_config()}, \texttt{check\_required\_packages()}
  \item \textbf{Lecture et audit} : \texttt{read\_delim\_auto()}, \texttt{audit\_csv\_conformity()}, \texttt{export\_csv\_conformity\_report()}
  \item \textbf{Pr\'eparation} : \texttt{prepare\_data()}, \texttt{deduplicate\_observations()}
  \item \textbf{Analyses univari\'ees} : \texttt{calc\_cumulative\_metrics()}, \texttt{calc\_tee\_ir()}, \texttt{calc\_species\_frequency()}, \texttt{calc\_spatial\_occupancy()}
  \item \textbf{Mod\'elisation} : \texttt{fit\_linear\_model()}, \texttt{fit\_hyperbolic\_model()}, \texttt{fit\_assymptote()}
  \item \textbf{M\'etriques avanc\'ees} : \texttt{calc\_scientific\_metrics()}, \texttt{calc\_ca\_analysis()}
  \item \textbf{Orchestration} : \texttt{analyze\_site()}, \texttt{run\_analysis()}, \texttt{log\_output\_manifest()}, \texttt{ensure\_thematic\_dirs()}
\end{itemize}

\noindent
Toutes ces clés supportent les surcharges via variables d'environnement \texttt{INVENTAIRES\_*}. La configuration est affichée dans l'en-t\^ete du script via la fonction \texttt{print\_startup\_header()}.

Variables d'environnement principales : \texttt{INVENTAIRES\_INPUT\_FILE},
\texttt{INVENTAIRES\_OUTPUT\_DIR}, \texttt{INVENTAIRES\_DATE\_FORMAT},
\texttt{INVENTAIRES\_CSV\_STRICT}, \texttt{INVENTAIRES\_MAKE\_CA} et
\texttt{INVENTAIRES\_AUTO\_RUN}. Les autres options reprennent les cl\'es de configuration.

\section*{Modes d'ex\'ecution et flags internes}
Le script expose des flags globaux :
\begin{itemize}
  \item \texttt{DEBUG\_MODE} (d\'efaut \texttt{FALSE}) : warnings affich\'es en continu et logs d\'etaill\'es.
  \item \texttt{BENCHMARK\_MODE} (d\'efaut \texttt{FALSE}) : chronom\'etrage des blocs de calcul.
  \item \texttt{CLEAN\_ENVIRONMENT} (d\'efaut \texttt{FALSE}) : purge de l'environnement avant lancement (mode expert).
\end{itemize}

Le lancement automatique est contr\^ol\'e via \texttt{INVENTAIRES\_AUTO\_RUN} (d\'efaut \texttt{TRUE}).

\section*{Contrat d'entr\'ee}
\texttt{data/observations.csv} (ou \texttt{.txt}/\texttt{.tsv}) avec colonnes obligatoires \texttt{site,date,visite\_id,espece} et optionnelle \texttt{placette}.

R\`egles : visite distincte \texttt{date + visite\_id} (et \texttt{site} en multi-sites), parsing date strict, d\'eduplication de la cl\'e observation.

\section*{Audit CSV}
Sorties d'audit :
\begin{itemize}
  \item \texttt{results/ICR/ICR\_00\_csv\_conformite\_report.csv}
  \item \texttt{results/ICR/ICR\_00\_csv\_conformite\_problems.csv} (si anomalies)
\end{itemize}

En mode strict, toute non-conformit\'e est bloquante.

\section*{Logging syst\'ematis\'e}
Chaque ex\'ecution g\'en\'ere un fichier log horodat\'e dans le r\'epertoire \texttt{logs/} du projet.

\subsection*{Fonctions de logging}
\begin{itemize}
  \item \texttt{setup\_logging(base\_dir, prefix)} : Initialise l'environnement .log\_env et crée le répertoire logs/ ; appelée au démarrage
  \item \texttt{log\_message(level, ...)} : Fonction centrale avec gestion intelligente des format strings
  \item \texttt{log\_info(...)} : Wrapper niveau INFO
  \item \texttt{log\_debug(...)} : Wrapper niveau DEBUG
  \item \texttt{log\_warning(...)} : Wrapper niveau WARN (affiche aussi en stderr)
  \item \texttt{log\_header(script\_name, script\_version)} : Affiche l'en-t\^ete du script avec métadonnées
  \item \texttt{log\_section(title)} : Affiche un titre de section avec délimiteurs
\end{itemize}

\subsection*{Format et contenu}
\begin{itemize}
  \item \textbf{Format des ent\'ees} : \texttt{[YYYY-MM-DD HH:MM:SS] [LEVEL] message}
  \item \textbf{Nom du fichier} : \texttt{logs/ICR\_YYYYMMDD\_HHMM.log}
  \item \textbf{Contenu} : En-t\^ete du script, métadonnées (timestamp, R version, working dir), progression du pipeline, manifeste final des livrables
\end{itemize}

\section*{Journalisation et int\'egrit\'e des sorties}
La journalisation est r\'ealis\'ee en console via \texttt{log\_info()}, \texttt{log\_debug()} et \texttt{log\_warning()}.

En fin de pipeline, \texttt{log\_output\_manifest()} contr\^ole les livrables avec statuts :
\texttt{OK}, \texttt{MANQUANT}, \texttt{OPTIONNEL\_NON\_GENERE}.

Le script supprime automatiquement tout \texttt{Rplots.pdf} parasite g\'en\'er\'e pendant l'ex\'ecution (niveau site et global).

\section*{Livrables}
\subsection*{Global}
\begin{itemize}
  \item \textbf{Métriques} (\texttt{06\_metrics/}) :
  \texttt{ICR\_donnees\_preparees.csv},
  \texttt{ICR\_resume\_tous\_sites.csv},
  \texttt{ICR\_metrics\_pertinence\_tous\_sites.csv}.

  \item \textbf{Comparaisons} (\texttt{comparisons/}) :
  \texttt{ICR\_comparaison\_completude\_sites.png},
  \texttt{ICR\_comparaison\_ir\_sites.png},
  \texttt{ICR\_metrics\_pertinence\_heatmap\_sites.png},
  \texttt{ICR\_metrics\_pertinence\_score\_sites.png}.
\end{itemize}

Audit CSV : rapport de conformité et liste des anomalies dans \texttt{00\_data\_quality/}.

Logs : \texttt{ICR\_YYYYMMDD\_HHMM.log} (dans \texttt{logs/}).

\subsection*{Par site (\texttt{sites/<site\_name>/})}
Fichiers CSV par site : courbe temps-espèces, TEE/IR, fréquences,
résumé de site et métriques de pertinence, avec les graphiques PNG associés.

Sorties conditionnelles : \texttt{ICR\_05\_modeles.csv}, \texttt{ICR\_06\_occupation\_spatiale.csv}, \texttt{ICR\_05\_temporel\_vs\_spatial.png}, \texttt{ICR\_06\_CA\_placettes\_especes.png}.

\section*{Flux d'ex\'ecution}
\begin{verbatim}
run_analysis(CONFIG)
  -> validate_config()
  -> resolve_input_file()
  -> read_delim_auto()
  -> audit_csv_conformity()
  -> export_csv_conformity_report()
  -> prepare_data()
  -> for each site: analyze_site()
       -> calculs cumul/TEE/frequences/spatial
       -> modeles lineaire/hyperbolique
       -> metriques scientifiques
       -> exports CSV/PNG
  -> consolidation multi-sites
  -> graphiques globaux
  -> log_output_manifest()
\end{verbatim}

\section*{D\'ependances}
Requises : \texttt{dplyr}, \texttt{tidyr}, \texttt{ggplot2}, \texttt{purrr}, \texttt{readr}, \texttt{stringr}, \texttt{forcats}, \texttt{tibble}, \texttt{scales}, \texttt{gridExtra}.\\
Optionnelles : \texttt{minpack.lm} (hyperbolique), \texttt{vegan} (CA/AFC).

\section*{Am\'eliorations techniques (v1.4)}
\subsection*{Phase 1 : Fiabilisation}
\begin{itemize}
  \item D\'etection automatique du d\'elimiteur (CSV/TSV/TXT)
  \item Parsing flexible des dates via format configurable
  \item D\'eduplication multi-cl\'e : \texttt{(site, placette, date, visite\_id, espece)}
  \item Gestion granulaire des erreurs CSV (colonne manquante, type invalide, etc.)
\end{itemize}

\subsection*{Phase 2 : Performance}
\begin{itemize}
  \item Optimisation du modèle hyperbolique : \texttt{minpack.lm::nlsLM} avec détection de convergence
  \item Calculs vectorisés par site (pas de boucle implicite)
  \item M\'emorisation des graphiques ggplot2 avant export PNG
  \item Cache des analyses de CA/AFC conditionnelle
\end{itemize}

\subsection*{Phase 3 : Graphiques et visualisations}
\begin{itemize}
  \item Cartes de chaleur multi-sites (heatmap) avec échelle de couleur
  \item Tableaux de bord par site (7 graphiques dans une grille)
  \item Comparaisons visuelles de complétude (barplot multi-sites)
  \item Analyses de CA/AFC si \texttt{vegan} disponible et placette présente
\end{itemize}

\subsection*{Phase 4 : Industrialisation}
\begin{enumerate}
  \item \textbf{Configuration centralis\'ee} : \texttt{get\_embedded\_config()} avec 14+ param\`etres
  \item \textbf{Logging syst\'ematis\'e} : \texttt{setup\_logging()} + fonctions log\_*() avec fichiers horodat\'es
  \item \textbf{Organisation th\'ematique} : hi\'erarchie logique /sites/, /06\_metrics/, /00\_data\_quality/, /comparisons/
  \item \textbf{M\'etriques avanc\'ees} : calcul de pertinence scientifique, dashboards analytiques, manifeste final
\end{enumerate}

\section*{Limites connues}
\begin{itemize}
  \item Sans \texttt{placette}, pas d'analyse spatiale.
  \item Sans \texttt{minpack.lm}, pas d'asymptote hyperbolique.
  \item Si \texttt{nb\_visites < min\_visits\_for\_model}, pas de mod\'elisation.
  \item En mode strict, un CSV non conforme bloque le pipeline.
\end{itemize}
