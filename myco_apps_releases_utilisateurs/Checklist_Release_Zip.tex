\subsection{Objectif}\label{objectif}

Cette archive fournit une interface locale pour lancer deux analyses R
sans ouvrir RStudio :

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  \textbf{Inventaires fongiques --- Complétude \& Représentativité
  (ICR)} ;
\item
  \textbf{Évaluation potentiel fongique / intérêt patrimonial / gradient
  CHEGD}.
\end{enumerate}

L'application s'ouvre dans votre navigateur, mais les calculs restent
exécutés sur votre ordinateur.

\subsection{Prérequis}\label{pruxe9requis}

Vous devez installer \textbf{R} une fois sur votre ordinateur :

\url{https://cran.r-project.org/}

RStudio n'est pas obligatoire.

\subsection{Lancement}\label{lancement}

\subsubsection{Windows}\label{windows}

Double-cliquez sur :

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{Lancer\_Windows.bat}
\end{Highlighting}
\end{Shaded}

\subsubsection{macOS}\label{macos}

Double-cliquez sur :

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{Lancer\_Mac.command}
\end{Highlighting}
\end{Shaded}
Si macOS bloque le lancement, faites clic droit puis \textbf{Ouvrir}.
Au premier lancement sur macOS, le système peut afficher : ``Apple n'a
pas pu confirmer que ce fichier ne contenait pas de logiciel
malveillant.''

\noindent Dans ce cas :

\begin{enumerate}
\item Cliquer sur \textbf{Terminé}.
\item Ouvrir Terminal dans le dossier de l'application.
\item Exécuter les commandes suivantes :
\end{enumerate}

\begin{Shaded}
\begin{Highlighting}[]
\NormalTok{xattr -dr com.apple.quarantine .}
\NormalTok{chmod +x Lancer\_Mac.command}
\NormalTok{./Lancer\_Mac.command}
\end{Highlighting}
\end{Shaded}

\subsubsection{Linux}\label{linux}

Dans un terminal :

\begin{Shaded}
\begin{Highlighting}[]
\ExtensionTok{./Lancer\_Linux.sh}
\end{Highlighting}
\end{Shaded}

\subsection{Premier lancement}\label{premier-lancement}

Au premier lancement, l'application installe automatiquement les
composants R nécessaires. Une connexion internet est donc nécessaire. Les lancements suivants sont normalement plus rapides.

\subsection{Utilisation}\label{utilisation}

\begin{enumerate}
\def\labelenumi{\arabic{enumi}.}
\tightlist
\item
  Choisir l'application à lancer.
\item
  Utiliser le fichier d'exemple ou charger votre propre fichier.
\item
  Cliquer sur \textbf{Lancer l'analyse}.
\item
  Consulter les fichiers produits dans l'onglet \textbf{Résultats} ou
  dans \texttt{applications/results/}.
\end{enumerate}

\subsection{Notes techniques}\label{notes-techniques}

Cette version ajoute une interface Shiny locale et des lanceurs multi-OS
autour des scripts existants. Le cœur des scripts métier est conservé.
Le script CHEGD a seulement été adapté pour accepter un fichier d'entrée
choisi depuis l'interface via variable d'environnement.

\subsection{Quitter l'application}\label{quitter-lapplication}

Dans l'interface, cliquez sur \textbf{Quitter l'application}.

Ensuite, fermez l'onglet du navigateur et, si elle reste ouverte, la
fenêtre Terminal.
